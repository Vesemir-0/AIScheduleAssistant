# AI Prompt 设计文档

日期：2026-03-09

## 概述

本文档定义了 AI Schedule Assistant 中用于识别图片和文本中日程事件信息的 AI Prompt 设计。

## 设计决策

### 核心需求
- **输入**：图片（截图）或文本
- **输出**：结构化的事件数据（JSON 格式）
- **目标**：准确识别日程、会议、任务和待办事项

### 关键决策
1. **日期时间格式**：AI 返回绝对日期时间（ISO 8601 格式）
2. **多事件处理**：识别所有事件，返回数组，用户在预览中选择
3. **分类匹配**：语义理解匹配现有分类
4. **优先级**：默认为 5（中等），用户可在预览中调整

## System Prompt 模板

```
你是一个专业的日程事件识别助手。你的任务是从图片或文本中识别所有的日程安排、会议、任务和待办事项，并将它们转换为结构化的 JSON 格式。

当前日期时间：{current_datetime}（ISO 8601 格式，如 2026-03-09T14:30:00+08:00）

用户的现有日历分类：{existing_calendars}
用户的现有待办清单分类：{existing_reminder_lists}

请识别所有事件，并为每个事件返回以下信息：
1. title：事件标题（简洁明了）
2. description：详细描述（可选）
3. startDate：开始日期时间（ISO 8601 格式）
4. endDate：结束日期时间（可选，ISO 8601 格式）
5. isAllDay：是否全天事件（布尔值）
6. dueDate：截止日期（ISO 8601 格式）
7. priority：优先级（默认 5）
8. suggestedCalendar：建议的日历分类
9. suggestedReminderList：建议的待办清单分类
10. confidence：识别置信度（0.0-1.0）

分类匹配规则：
- 理解事件的语义和现有分类的含义
- 优先匹配现有分类
- 如果没有合适的现有分类，建议创建新分类（遵循大类原则：工作、生活、学习、健康等）
- 避免创建过于细分的分类

日期时间处理规则：
- 所有日期时间必须转换为绝对时间（ISO 8601 格式）
- 基于当前日期时间计算相对日期（如"明天"、"下周一"）
- 如果没有指定时间，默认使用 09:00
- 如果没有指定日期，使用当前日期

返回格式：
{
  "events": [
    {
      "title": "事件标题",
      "description": "详细描述",
      "startDate": "2026-03-10T15:00:00+08:00",
      "endDate": "2026-03-10T16:00:00+08:00",
      "isAllDay": false,
      "dueDate": "2026-03-10T15:00:00+08:00",
      "priority": 5,
      "suggestedCalendar": "工作",
      "suggestedReminderList": "工作",
      "confidence": 0.95
    }
  ]
}
```

## User Prompt 模板

### 图片识别
```
请识别这张图片中的所有日程事件、会议、任务和待办事项。
```

### 文本识别
```
请识别以下文本中的所有日程事件、会议、任务和待办事项：

{user_text}
```

## 实现要点

### 动态参数注入
- `{current_datetime}`：当前系统日期时间
- `{existing_calendars}`：从 EventKit 读取的现有日历列表
- `{existing_reminder_lists}`：从 EventKit 读取的现有待办清单列表
- `{user_text}`：用户选中的文本内容

### 图片处理
- 将 NSImage 转换为 base64 编码
- 使用 Vision API（如果支持）或直接发送给 AI

### 响应解析
- 解析 JSON 响应
- 验证必填字段
- 转换 ISO 8601 字符串为 Date 对象
- 处理解析错误和降级策略

### 错误处理
- API 调用失败
- JSON 解析失败
- 日期格式错误
- 置信度过低（< 0.5）

## 测试用例

### 测试用例 1：简单会议
**输入文本**：明天下午3点开会
**预期输出**：
- title: "开会"
- startDate: 2026-03-10T15:00:00+08:00
- endDate: 2026-03-10T16:00:00+08:00
- isAllDay: false

### 测试用例 2：多个事件
**输入文本**：
```
1. 明天上午10点团队会议
2. 下周一提交报告
3. 3月15日体检
```
**预期输出**：3个事件的数组

### 测试用例 3：全天事件
**输入文本**：3月20日休假
**预期输出**：
- isAllDay: true
- startDate: 2026-03-20T00:00:00+08:00

## 优化方向

### 短期优化
- 添加更多示例到 prompt 中（few-shot learning）
- 优化日期时间识别的准确性
- 改进分类匹配算法

### 长期优化
- 支持重复事件识别
- 支持多语言
- 学习用户的分类习惯
- 支持更复杂的时间表达（如"每周三"）
