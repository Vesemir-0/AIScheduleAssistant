# AI Schedule Assistant - 测试指南

本指南将帮助你一步步测试应用的所有功能。

## 准备工作

### 1. 构建应用

1. 打开 Xcode
2. 打开项目文件：`AIScheduleAssistant.xcodeproj`
3. 选择目标设备：`My Mac`
4. 点击运行按钮（▶️）或按 `Cmd + R`
5. 等待编译完成

**预期结果**：
- 应用成功启动
- 菜单栏右上角出现日历图标 📅

### 2. 首次启动

**步骤**：
1. 应用启动后会自动弹出欢迎窗口

**预期结果**：
- 看到欢迎窗口，显示需要的 4 个权限
- 有"请求权限"和"稍后设置"两个按钮

**测试点**：
- ✅ 欢迎窗口正常显示
- ✅ 权限列表清晰可见
- ✅ 按钮可以点击

## 第一部分：权限管理测试

### 测试 1.1：请求权限

**步骤**：
1. 在欢迎窗口点击"请求权限"
2. 系统会依次弹出权限请求对话框

**预期结果**：
- 
- 日历权限：弹出系统对话框，点击"好"授权
- 提醒事项权限：弹出系统对话框，点击"好"授权
- 屏幕录制权限：自动打开系统设置页面

**操作**：
1. 对于屏幕录制：
   - 在系统设置中找到应用
   - 勾选启用
2. 对于日历和提醒事项：
   - 直接点击"好"授权

### 测试 1.2：查看权限状态

**步骤**：
1. 点击菜单栏图标
2. 选择"设置"
3. 切换到"权限管理"标签页

**预期结果**：
- 看到 4 个权限的状态
- 已授权的显示绿色 ✅
- 未授权的显示红色 ❌

**测试点**：
- ✅ 权限状态正确显示
- ✅ "刷新状态"按钮可用
- ✅ "打开系统设置"按钮可以跳转

## 第二部分：配置 AI API

### 测试 2.1：配置 OpenAI API

**步骤**：
1. 点击菜单栏图标
2. 选择"设置"
3. 在"AI 配置"标签页中填写：
   - API 端点：`https://api.openai.com/v1`
   - API Key：你的 OpenAI API Key
   - 模型名称：`gpt-4` 或 `gpt-3.5-turbo`
   - Temperature：保持默认 0.7
   - Max Tokens：保持默认 1000
4. 点击"保存"

**预期结果**：
- 配置保存成功
- 没有验证错误提示

**测试点**：
- ✅ 所有字段可以输入
- ✅ API Key 字段显示为密码（不可见）
- ✅ 保存按钮可用
- ✅ 配置验证通过

### 测试 2.2：配置其他 AI 服务（可选）

如果你使用其他 AI 服务（如 Claude、国内大模型等）：

**示例 - 使用 Anthropic Claude**：
- API 端点：`https://api.anthropic.com/v1`（需要兼容层）
- 模型名称：`claude-3-opus-20240229`

**示例 - 使用本地 Ollama**：
- API 端点：`http://localhost:11434/v1`
- 模型名称：`llama2`

## 第三部分：截图识别测试

### 测试 3.1：简单日程识别

**准备**：
1. 打开一个文本编辑器（如备忘录）
2. 输入测试文本：
   ```
   明天下午3点团队会议
   ```

**步骤**：
1. 点击菜单栏图标
2. 选择"截图获取"
3. 系统进入截图模式（十字光标）
4. 框选包含文本的区域
5. 松开鼠标完成截图

**预期结果**：
- 应用开始分析（可能需要几秒钟）
- 弹出预览窗口
- 显示识别的事件：
  - 标题：团队会议
  - 日期：明天的日期
  - 时间：15:00
  - 分类：工作（或其他合理分类）
- 事件默认被选中（✓）

**测试点**：
- ✅ 截图功能正常
- ✅ AI 正确识别日期（明天）
- ✅ AI 正确识别时间（下午3点 = 15:00）
- ✅ 预览窗口正常显示
- ✅ 事件信息完整

### 测试 3.2：多个事件识别

**预期结果**：

- 底部显示"3 / 3 个事件已选择"

**测试点**：
- ✅ 识别出所有 3 个事件
- ✅ 日期正确（周一、周三、周五）
- ✅ 时间正确
- ✅ 可以单独选择/取消选择事件

### 测试 3.3：全天事件识别

**准备**：
```

```

**预期结果**：

## 常见问题排查

### 问题 1：应用无法启动

**检查**：
- Xcode 是否显示编译错误
- macOS 版本是否 >= 14.6
- 是否有代码语法错误

### 问题 2：权限请求失败

**检查**：
- 系统设置 > 隐私与安全性
- 确认应用出现在权限列表中
- 尝试手动授权

### 问题 3：AI 识别失败

**检查**：
- API Key 是否正确
- 网络连接是否正常
- API 端点是否正确
- 查看 Xcode 控制台的错误日志

### 问题 4：事件未创建

**检查**：
- 日历和提醒事项权限是否授予
- 打开日历/提醒事项应用刷新
- 检查是否有错误通知
